C: Tabbed Preview

Language tabs at the top, ADT grid immediately visible below. Switching tabs changes the ADT names/methods. Feels like the current home page with a language toggle added.

Recommendation: B

  • Gives enough context to make an informed choice
  • Shows what's different between tracks (JShell vs Pyodide)
  • Natural entry point that scales to more languages later
  • After clicking, users land on the familiar ADT grid (existing HomePage pattern)

Trade-offs

  • A is faster if users already know what they want
  • C keeps everything on one page but gets crowded with two language datasets
